--- layout: linked_pages/specification title: Zone de décontamination slug: decontamination type: functional workflow: - content: Les fichiers sont déposés dans une zone spécifique, appelée « Guichet A ». - content: Une première analyse est effectuée sur le « Guichet A ». - content: Le fichier est récupéré vers une zone de partage intermédiaire pour une deuxième analyse. - content: Le fichier sera enfin transféré vers une zone de partage final, appelée « Guichet B » pour une dernière analyse. - content: Lors d'une analyse, si le fichier est déclaré « illicite » ou « dangereux », il est placé en quarantaine ou supprimé. - content: Si le fichier est déclaré « sain », il est mis à disposition dans la zone de partage final. - content: Les utilisateurs peuvent récupérer les fichiers depuis le « Guichet B ». ---

Le projet doit définir la mise en place d'un système de décontamination de fichiers à travers une analyse croisée afin de garantir la licéité des fichiers transmis et réceptionnés avant de les rendre disponibles pour les utilisateurs.

À noter que les analyses doivent prendre en charge les formats courants : « .pdf », « .docx », « .xlsx », « .jpg », « .zip », etc.

Restriction des accès

Les machines doivent communiquer le moins possible avec l'extérieur, avec Internet ou avec d'autres services. Il faudra définir les paramétrages à appliquer afin de limiter les communications sur l'infrastructure cible.

Le processus entier doit être automatisé pour garantir une réactivité et une efficacité maximale.
